Klaar Help

Klaar is a collaborative todo list application. It supports nested items, tags, filtering, views, sharing, and real-time sync across devices.

Getting Started

Klaar organizes your work into lists, each containing items. Items can be nested to form hierarchies, tagged for categorization, and filtered using text search, tags, or completion status. You can save filter and sort configurations as views for quick access.

The interface has three panels:

  • Lists panel (left) — shows all your lists and shared lists. Toggle with the hamburger button.
  • Items area (center) — shows the items of the selected list, along with the search bar and completion filter.
  • Tags and Views panel (right) — shows tag definitions and saved views for the current list. Toggle with the dots button.

Items

Creating items

Press Enter in an item's text field to create a new item below it at the same depth. If the cursor is at the very beginning of a non-empty item, the new item is inserted above instead. When a list is first opened with no items, an empty item is created automatically.

Press Shift+Enter to create a new item that inherits the same tags (without values) as the current item.

Editing items

On desktop, each item is an editable text field. Changes are saved automatically when you move to another item or click away. On mobile, tap the item text to open an inline editor.

Deleting items

Click the x button on the right side of an item to delete it. Press Backspace on an empty item to delete it and focus the previous item.

Hold Ctrl and click the x button to delete the item and all its children.

Hierarchy and nesting

Items can be nested to create parent-child hierarchies. Press Tab to indent (increase depth) and Shift+Tab to outdent (decrease depth). Maximum nesting depth is 20 levels.

Parent items show a collapse/expand triangle in the left gutter. Click it or press Ctrl+. to toggle. Collapsed parents show a child count badge.

The depth bar above the items area provides buttons to collapse or expand all items at a given depth level. Click a depth number to collapse all items at that depth (expanding items above). Click it again to expand all. The "All" button expands everything.

Checkboxes

Each item has a checkbox to mark it as done. Completed items are displayed with strikethrough text.

Hold Ctrl and click a checkbox to toggle done status for the item and all its children.

If multiple items are selected, clicking any selected item's checkbox toggles done status for the entire selection.

Hover over a completed item's checkbox on desktop to see when it was marked done.

Context menu

Right-click an item (or long-press on mobile) to open its context menu. Options include:

  • Add above / Add below — insert a new item adjacent to the target.
  • Move / Move with children — on mobile, enter reorder mode to re-position the item (or the item plus its children) without drag-and-drop.
  • Delete — remove the item (and if multiple items are selected, the whole selection).
  • Copy to clipboard — copy the item (or selection) as indented Markdown-style text.
  • Visit link — if the item's text contains a URL, open it in a new tab.
  • Gather matching — find sibling items with the same text as this one and merge their children under this item. Useful for consolidating duplicate headings.
  • Indent / Outdent — adjust depth (keyboard-less alternative to Tab).
  • Select to here — extend the current selection to include every item between the last-selected one and this one.
  • Tag checklist at the bottom — toggle tags on the target and set tag values; see the Tags section for details.

Resize the text column (desktop)

A thin ruler above the items area has a draggable marker. Drag it left or right to change the width of the item text column; tags wrap to the next line if the text pushes past the marker. The chosen width is remembered per list.

Selection

  • Click an item to select it (clears previous selection).
  • Ctrl+click to toggle an item in or out of the selection.
  • Shift+click to select a range from the last-selected item to the clicked item.

Selected items are highlighted and exempt from filtering — they remain visible even if they do not match active filters.

Drag and drop reordering

Click and drag an item to reorder it within the list. The item moves in real time as you drag.

  • Ctrl+drag moves the item along with all its children.
  • Dragging with multiple items selected consolidates and moves the entire selection.
  • Drag an item onto a list name in the sidebar to move it to that list. Hover over a different list for one second to switch to that list while dragging, letting you position the item precisely.

Tags

Tags let you categorize and annotate items. Each list has its own set of tag definitions, shown in the Tags panel on the right.

Creating tags

Click the + button in the Tags panel header. Type a name and press Enter to create the tag. Each tag is assigned a color automatically.

Applying tags to items

  • From the tag pane: Select or focus one or more items, then click a tag name in the tag pane to toggle it on/off. Hold Ctrl while clicking the tag name to also apply to children of the selected items.
  • From the context menu: Right-click an item (or long-press on mobile) and use the tag checklist at the bottom of the context menu.
  • Removing a tag: Click a tag bubble on an item to remove it. Hold Ctrl while clicking the bubble to remove it from the item and all its children.

Tag values

Tags can have optional values. Double-click a tag bubble on an item (or use the value field in the context menu's tag checklist) to edit. Values can be any text, number, or date.

Setting a value on multiple items at once: if several items are selected and you edit a tag value on any one of them, pressing Enter spreads the new value to every selected item that has the same tag, overwriting any existing values. Press Shift+Enter instead to fill only the selected items where the tag currently has no value. Clicking away (blur) commits the value to just this one item without spreading. Escape aborts without saving.

Date values

If a tag value is a date (YYYY-MM-DD format), it is displayed as a friendly label such as "Today", "Tomorrow", "This Wednesday", or "Jan 15". Hover over the bubble to see the raw date in a tooltip.

Double-clicking a tag bubble whose value is already a date auto-opens a date picker. You can also press Ctrl+D while editing a tag value to open the picker manually. Picking a date counts as an explicit commit — it spreads to all selected items the same way Enter does.

Tag colors

Click the colored dot next to a tag name in the tag pane to change its color using a color picker.

Renaming tags

Double-click a tag name in the tag pane to rename it.

Hiding and showing tags

Each tag has a visibility toggle (filled/hollow circle) in the tag pane. Hidden tags are not displayed as bubbles on items, reducing visual clutter. The tag still exists and can still be used for filtering.

Sorting by tag value

Click the sort button (up/down arrows) next to a tag to sort items by that tag's value. Click again to reverse direction, and a third time to clear the sort. Items without the tag sort to the end. Sorting operates on top-level blocks — children stay grouped with their parent.

Reordering tags

Drag a tag entry in the tag pane to reorder the tag list. This affects the order in which tag bubbles appear on items.

Deleting tags

Click the x button next to a tag to delete it. This removes the tag from all items in the list.

Views

Views save the current filter, sort, completion filter, and tag visibility state so you can quickly switch between different perspectives on your list.

Saving a view

Set up your desired filters, sort order, and hidden tags, then click the + button in the Views section of the tag pane. Type a name and press Enter. The new view captures the current state of:

  • Text filters
  • Tag filters (include/exclude, with conditions)
  • Completion filter (All / Active / Done)
  • Date filter (📅 toggle)
  • Sort order — either a tag sort (tag + direction) or the virtual date sort
  • Hidden tag IDs

Applying a view

Click a view name to apply it. The active view is highlighted. Click it again to toggle it off and clear all filters.

Default view (auto-restore)

When you apply a view, it is saved as the list's active view. The next time you open that list, the view is automatically restored.

Overwriting a view

Click the refresh button next to a view to overwrite it with the current filter/sort state.

Renaming a view

Double-click the view name to rename it.

Deleting a view

Click the x button next to a view to delete it.

Filtering

Filtering lets you focus on specific items. Multiple filters can be combined and are applied together (all must match).

Text search

Type in the search bar to highlight matching items in real time. The search supports regular expressions. Press Enter to commit the search as a persistent text filter (shown as a bubble below the search bar). Press Escape to clear the search input.

Multiple text filters can be stacked — each one narrows the results further. Click a filter bubble to remove it.

Tag filters

Click the filter button (triangle icon) next to a tag in the tag pane to add a tag filter. This shows only items that have that tag.

Tag filter bubbles appear in the filter bar:

  • Click a tag filter bubble to toggle between include mode (filled, showing items with the tag) and exclude mode (outlined, hiding items with the tag).
  • Double-click a tag filter bubble to set a condition (e.g., >5, =Alice, <=2026-01-01). Conditions support =, !=, >, <, >=, <= operators, or plain text for substring matching.
  • Click the x on a filter bubble to remove it.

Multiple filters for the same tag can be added for complex conditions.

Completion filter

The three buttons next to the search bar filter by completion status:

  • All — show all items.
  • Active — show only uncompleted items.
  • Done — show only completed items.

Date filter (📅)

The calendar button next to the completion filter toggles a tag-agnostic date filter: only items with at least one date-valued tag are shown, sorted by the item's earliest date (oldest first). If you had a different sort active when you enabled the date filter, that sort is remembered and restored when you disable the filter. Picking a different tag sort while the date filter is on overrides the memory — your deliberate choice wins.

Ancestor visibility

When filters are active, parent items that do not match the filter but have matching descendants are shown with a muted style, preserving the hierarchy context.

Selected items

Selected items are always visible regardless of filters. This allows you to keep working with selected items even when a filter would normally hide them.

Upcoming Dated Items

The calendar button (📅) in the Lists panel header opens an Upcoming modal: a cross-list, read-only snapshot of every not-done item that has a date-valued tag, grouped by day.

Items are pulled from every list you can access — your own and shared — and sorted earliest-date first. Each entry shows the item text, its date tags, and a breadcrumb of its ancestor items and list name so you can tell where it lives.

By default the snapshot spans from one week ago to one week ahead. Use the "Next" selector in the header to extend the forward window (up to 3 months). A "Load earlier items" button at the bottom adds any older overdue items.

Click an item to jump to it in its list (expanding any collapsed ancestors). The refresh button (↻) re-fetches in case lists changed while the modal was open.

Lists

Creating a list

Click the + button in the Lists panel header. Type a name and press Enter. The new list is created and selected automatically.

Renaming a list

Double-click a list name in the sidebar, or edit the list title at the top of the items area. You can also right-click (or long-press on mobile) and choose "Rename" from the context menu.

Deleting a list

Click the x button next to a list name in the sidebar, or right-click and choose "Delete". A confirmation dialog is shown before deletion.

Reordering lists

Drag and drop list names in the sidebar to reorder them. The order is saved per user.

Import and export (⇅)

The transfer button (up/down arrows) in the Lists panel opens a small popover with both options:

  • Import from text — paste text to create items. The importer handles indentation (spaces or tabs) for nesting depth, bullet prefixes (-, *, numbered lists), checkboxes ([x] for done, [ ] for active), and Markdown strikethrough. You can import as a new list or add to the current list.
  • Export as Markdown — downloads the current list as a Markdown file with indentation, checkboxes, and tag annotations preserved.

List context menu

Right-click a list (or long-press on mobile) for these actions:

  • Rename, Delete (owner only), Leave list (non-owners).
  • Sharing — opens the sharing dialog (see Sharing).
  • Copy calendar URL — copies a list-scoped ICS subscription URL to your clipboard (see Calendar Subscription).

Sharing and Collaboration

Lists can be shared with other users for real-time collaboration.

Permission levels

  • None — no access (removes sharing).
  • View — can see the list and its items, but cannot make changes.
  • Check — can view and toggle item checkboxes, but cannot edit text, create/delete items, or modify tags.
  • Edit — full read/write access to items, tags, and list settings.

Sharing a list

Right-click a list (or long-press on mobile) and choose "Sharing" to open the sharing dialog. You must add contacts on the User settings page before you can share. In the dialog, select a permission level for each contact.

Contacts

Contacts are managed on the User settings page. Send a contact request by username. When the other user accepts, you can share lists with them.

Shared list indicators

Lists shared with you display a people icon next to the list name in the sidebar. The list owner is shown in the sharing dialog.

Leaving a shared list

If a list was shared with you by someone else, you can leave it by clicking the x next to the list name in the sidebar, or via the right-click context menu. You will lose access unless the owner shares it again.

Real-time sync

Changes made by other users are synced automatically via polling. The poll interval starts at 5 seconds and gradually slows to 30 seconds when no changes are detected. It speeds up again when activity is detected. Polling pauses when the browser tab is hidden and resumes immediately when you return.

Calendar Subscription

Klaar exposes your dated items as iCalendar (.ics) feeds. Point any calendar app (Google Calendar, Apple Calendar, Outlook…) at a feed URL and your Klaar items show up as all-day events alongside your other calendars. Feeds are read-only — edits in the calendar don't flow back to Klaar.

Two URL flavors

  • All lists — shown on the User settings page under "Calendar Subscription". Includes every dated item you can see, across all your own and shared lists.
  • Single list — right-click any list in the sidebar and choose "Copy calendar URL" to put a list-scoped URL on your clipboard.

Both URL forms share a single secret token per user. Regenerating the token (from the User page) invalidates every subscription at once — the all-lists URL and every per-list URL — so only regenerate if you need to revoke access.

What appears in the calendar

Each not-done item with a date-valued tag produces one all-day event per date tag. The event title is <item text> (<tag name>) so items with multiple dated tags (e.g. due and reminder) show which date is which. Done items are excluded. Deleting an item or marking it done in Klaar removes the event on the next sync.

Aligned hierarchies collapse: if a child item shares the same date-tag value as one of its ancestors, the child isn't emitted as its own event. Instead, it's listed under the top-most matching ancestor's event description. A project with three subtasks all marked due: 2026-05-01 shows up in the calendar as a single event on that date, with the subtasks listed inside.

Each event also carries a link back to the specific Klaar item — your calendar app typically renders it in the event detail as a clickable URL. On desktop this opens Klaar and jumps straight to the item (expanding any collapsed ancestors and scrolling into view). On mobile, if Klaar is installed as a home-screen web app, the link may open inside the app; otherwise it opens in a new browser tab.

Sync latency

Google Calendar polls external feeds on its own schedule, typically every 6–12 hours (sometimes longer) — this is the one real downside of the iCal approach and Google does not let you shorten it. Apple Calendar and Outlook are more aggressive and let you set the interval (often down to 15 minutes).

Subscribing

  • Google Calendar: Left sidebar → "Other calendars" → "+" → "From URL" → paste.
  • Apple Calendar: File → New Calendar Subscription → paste.
  • Outlook: Add calendar → Subscribe from web → paste.

API Access

Each list can have its own API token — a per-list secret that lets an external tool (most usefully an AI assistant like Claude) read and edit the items in that one list, without giving the tool access to your account or any other list.

When to use it

The driving use case: you've worked out a plan or to-do structure with a chatbot, and rather than copying it line-by-line into Klaar, you want the chatbot to write the items into your list directly. Generate a token, paste it into the chat with a brief instruction, and it can do the work for you.

Generating a token

  • Right-click the list in the sidebar (long-press on mobile) and choose "API access…".
  • If the list has no token yet, click Generate token. The endpoint URL and token will appear.
  • Use the Copy share message button to put a ready-to-paste blurb on your clipboard — it includes the endpoint, the token, the auth header to use, and a link to the API reference. Drop it into the chat and the assistant has everything it needs.
  • Alternatively, use Copy URL and Copy token separately if you'd rather assemble the message yourself.

What the token can and can't do

The token grants full edit access on the one list it belongs to: read items and tags, add/edit/delete/reorder items, including bulk operations. It does not grant access to:

  • Other lists you own or are shared on.
  • Your account, password, contacts, or settings.
  • List metadata changes (rename, delete, sharing, tag definitions).

If the token is leaked, the worst case is unwanted edits to that one list — recoverable via undo and by revoking the token.

Rotating and revoking

  • Regenerate — from the same "API access…" dialog, issues a new token. The old token stops working immediately.
  • Revoke — clears the token entirely; the list goes back to having no API access until you generate a new one.
  • The User settings page has a List API Tokens section listing every list with an active token, with creation and last-used timestamps and a per-row revoke button. Useful for spotting tokens you forgot about.

API reference

The API reference documents endpoint shapes (item schema, depth-based hierarchy, bulk operations). The "Copy share message" button gives the AI assistant a link to it directly — you don't need to read it yourself unless you want to script against the API by hand.

Dark Mode

Click the sun/moon button in the header to toggle between light and dark modes. Your preference is saved in the browser.

If no preference has been set, Klaar follows your operating system's color scheme preference automatically.

User Account

Click your display name in the header to access the User settings page.

  • Display name — change how your name appears to others.
  • Password — change your password (requires current password).
  • Contacts — send contact requests, accept or decline incoming requests, and manage your contact list.
  • Calendar subscription — copy your all-lists ICS URL or regenerate the token. See Calendar Subscription.
  • List API tokens — review which lists have active API tokens and revoke any you no longer need. See API Access.
  • Delete account — permanently delete your account and all your owned lists.

Admin users also see a user administration section where they can create, edit, and delete other users.

Mobile

Klaar adapts to smaller screens with a mobile-optimized layout.

Install as a web app (recommended)

Klaar works noticeably better on phones when installed to the home screen as a web app — the browser address bar and chrome go away, giving more vertical space and a more native feel. On iOS, open Klaar in Safari, tap the Share button, and choose "Add to Home Screen". On Android, open Klaar in Chrome, tap the menu (⋮), and choose "Add to Home screen" (or "Install app").

Panels

On mobile, the Lists and Tags panels slide in as overlays from the left and right edges. Tap outside a panel or use the toggle buttons to close them.

Touch interactions

  • Tap an item's text to open an inline editor.
  • Long-press (500ms) an item to open the context menu as a bottom sheet.
  • Long-press a list name in the sidebar to open the list context menu.
  • Swipe right / left on a selected item to indent / outdent it (replaces Tab/Shift+Tab on touch). If multiple items are selected, all of them move. The gesture has to be clearly horizontal to engage, so you won't accidentally trigger it while scrolling.

Context menu

On mobile, the context menu appears as a bottom sheet rather than a floating menu. See the Items → Context menu list for what's in it.

Reorder mode

Since drag and drop is not available on mobile, use the context menu's "Move" or "Move with children" option to enter reorder mode. A cursor line appears at the center of the screen. Scroll the list to position the cursor where you want the item, then tap "Place here" to confirm or "Cancel" to abort.

Debug panel

Long-press the "Klaar" title in the header to toggle an on-screen debug log, useful for troubleshooting mobile-specific issues.
